Ditt CV

På den här webbplatsen kan du lagra ditt CV i form av anställningar med Företagsnamn, arbetstitel, ort, start och sluttid för anställningen samt en längre beskrivning av arbetet. Inläggen sparas i en mongoDB-databas med hjälp av ett api. Inläggen ligger synligt för alla som använder sidan och skriv därför inte in känslig information.

Uppgift

Webbplatsen är en del av moment3 i kursen backend-baserad webbutveckling som går ut på att skapa ett api i en expresserver som kan hantara Fetch-anrop med get, post, put och delete. Denna api ska också kunna skicka och hämta data från en mongoDb-databas och skicka vidare svaren till denna webbplats. Så här har jag löst det:

  • Jag har skapat en MariaDB-databas som är hostad hos HelioHost.
  • Node.js, expresserver hostad hos Azure som vid Fetch-anrop skickar frågor till databasen och svarar webbplatsen. Tillägget mongoose används i expresserver för anslutning till mongoDb.
  • Denna webbplats är hostad hos Netlify för att visa funktionen hos api och som skickar anropen get för att hämta cv, post för att skapa nya från formuläret och delete för att ta bort enskilda inlägg. Utöver den funktionaliteten kan api också hantera put för att redigera ett helt inlägg eller enskilda värden.

En mer ingående beskrivning av api-funktionerna och hur den används hittar du här.

Sammanfattning

Det känns som användbart med apier och det går smidigt att på ett säkert sätt nå databaser. Att skapa lösningen för mongoDb-databasen var enklare än att göra samma uppgift med mySQL. Eftersom att moment2 av denna uppgift använde likadan frontend lösning. Jag ändrade bara i en JavaScript-loop för att skriva ut på startsidan så att rätt namn användes. Mongoose var också enkelt att arbeta med. Apin är publicerad på Azure och mongoDb-databasen ligger på Atlas. Atlas och mongoDb har varit en enkel och bra upplevelse att arbeta med. Allt som allt en lärorik upplevelse.
